# PRD：家庭记账桌面应用（v2）

## 一、项目信息

- **项目名称**：family_accounting_app
- **语言**：中文
- **目标平台**：macOS 桌面应用（v2 约束：技术选型需兼顾未来 Windows 版本复用）
- **技术栈**：跨平台方案（见待确认问题 #1），数据存储 SQLite（已确认）
- **原始需求复述**：用户使用 Excel 记账 8 年（2018-2026），已跑通固定的月度记账流程（导出微信/支付宝账单 → 清洗 → 分类 → 汇总 → 报表）。痛点是每次手动操作太累，希望将固定流程固化为桌面应用，能自动化的自动化、需要人工介入时提示介入，最终节省时间。v2 在 v1 基础上新增 AI 辅助分类/分析/报表、汇总下钻、Excel 级编辑等能力。

---

## 二、产品目标

**一句话目标**：将用户已成熟的 Excel 月度记账流程固化为桌面应用，实现账单导入、自动分类、AI 辅助、月度汇总、报表呈现的全流程自动化，把每月手工操作时间从数小时压缩到分钟级。

**三个正交目标**：
1. **流程自动化**：账单导入 → 清洗 → 分类 → 汇总 → 报表 全链路自动化，人工仅在关键节点介入确认
2. **AI 增效**：利用大模型辅助未匹配交易分类、月度消费分析、报表生成，进一步降低人工负担
3. **跨平台复用**：技术选型兼顾 macOS 与未来 Windows 版本，核心逻辑可复用

## 三、用户故事

1. **作为家庭记账人**，我希望能在月初导入上月的微信和支付宝账单文件，导入时选择账单归属人（joey/vila）自动填充支付人字段，以便不用手动复制粘贴数据到总账。

2. **作为家庭记账人**，我希望点击"清洗"按钮后系统过滤掉无效交易，并让我确认被过滤的数据是否符合删除标准，以便账单数据干净准确且不误删。

3. **作为家庭记账人**，我希望系统能根据分类规则自动给每笔交易打上消费标签，未匹配的先由 AI 大模型辅助判断再由我确认，以便减少人工分类工作量。

4. **作为家庭记账人**，我希望能在汇总表中手动录入收入、房贷/存钱、理财、保险等非账单数据，以便月度汇总完整。

5. **作为家庭记账人**，我希望在汇总页面点击某分类金额（如"家用 1100"）能跳转到明细页面并自动筛选对应子标签，以便快速查看明细。

6. **作为家庭记账人**，我希望点击"AI 分析"按钮对当月消费进行分析，并查看 AI 生成的 HTML 报表，以便快速了解收支状况和趋势。

7. **作为家庭记账人**，我希望能像 Excel 一样方便地编辑数据（修改分类、金额、标记等），以便保持操作效率。

8. **作为家庭记账人**，我希望能新增和调整分类规则（支持等于/包含/匹配等条件），以便应对新的消费场景。

## 五、账单文件格式规范（基于真实 demo 文件）

### 微信账单（XLSX 格式）

- **文件格式**：XLSX（非 CSV）
- **文件名示例**：`vila微信支付账单流水文件(20260601-20260630).xlsx`
- **文件结构**：前 17 行头部信息，第 18 行表头，第 19 行起为数据
- **头部信息**：微信昵称（如 `[Vila]`）、起止时间、导出时间、记录统计（收入/支出/中性交易笔数）、说明
- **表头**：交易时间, 交易类型, 交易对方, 商品, 收/支, 金额(元), 支付方式, 当前状态, 交易单号, 商户单号, 备注
- **收/支字段值**：支出、收入
- **当前状态值**：已转账、支付成功、已存入零钱、已全额退款
- **交易时间**：datetime 类型，精确到秒（如 `2026-06-30 16:51:40`）
- **特殊说明**：头部提到"中性交易"概念（充值/提现/理财通购买/零钱通存取/信用卡还款），收/支字段不单独标注

### 支付宝账单（CSV，GBK 编码）

- **文件格式**：CSV，**GBK 编码**
- **文件名示例**：`vila支付宝交易明细(20260601-20260630).csv`
- **文件结构**：前 22 行头部信息，第 23 行表头，第 24 行起为数据
- **头部信息**：姓名（如"赵紫薇"）、支付宝账户（如 `15730284920`）、起止时间、记录统计（含收入/支出/不计收支笔数和金额）、特别提示
- **表头**：交易时间, 交易分类, 交易对方, 对方账号, 商品说明, 收/支, 金额, 收/付款方式, 交易状态, 交易订单号, 商家订单号, 备注
- **收/支字段值**：支出、收入、**不计收支**（如余额宝收益发放、充值提现、账户转存）
- **交易状态值**：交易成功、交易关闭
- **不计收支处理**：导入并保留，标记为"不计收支"，不参与收支汇总计算
- **格式稳定性**：已确认，只要微信/支付宝不修改导出格式，导入格式不会变化，无需特别容错

## 六、核心流程描述

APP 主流程围绕"每月一次"的记账周期展开：

```
① 导入账单（选归属人）
   用户在月初打开 APP → 分别导入微信账单(XLSX)和支付宝账单(CSV/GBK)
   → 导入时选择账单归属人（joey/vila）（非必填）→ 自动填充支付人字段（如果用户没给就给空）
   → 支持多月导入（文件可含多月数据）
   → 系统解析文件，跳过头部行，提取有效字段
   → 把账单明细按 汇总明细表 字段进行录入。原始导入表保留可溯源。

② 数据清洗（按钮+确认）
   用户点击"数据清洗"按钮 
   → 系统过滤【还款成功】【交易关闭】【退款成功】【不计收入】状态交易
   → 系统过滤【金额】小于等于 3 的交易
   → 对于【部分退款】的交易，如果退款部分的金额小于等于 3，就将这条交易的状态改为 【支付成功】
   → 对于【部分退款】的交易，如果退款部分的金额大于 3，就将这条交易的状态改为【部分退款】，且将支付金额减去退款金额，作为最终的金额值
   → 列出将被过滤以及修改的数据 → 用户确认 → 执行过滤和修改

③ 自动分类 + AI 辅助
   系统根据分类规则引擎，对每笔交易匹配 交易对方/商品/交易类型
   → 自动填入消费标签（房租、买菜、餐饮 等）
   → 未匹配规则的交易 → AI 大模型辅助判断分类
   → AI 判断结果 + 用户修正结果 → 记忆学习，逐步提升准确率
   → 仍无法匹配的交易进入"待人工复核"队列

④ 人工介入
   用户查看待复核队列 → 逐条指定标签
   → 用户可修正任意已分类交易（Excel 级编辑体验）
   → 用户标记是否刚需、修正支付人等
   → 用户可标记特殊交易删除（从汇总表删，原始表留）
   → 消费标签字段非必填字段，如果最后无法分类，可以置空

⑤ 手动数据补充
   用户手动录入当月非账单数据：Joey/Vila收入、房贷/存钱、理财、保险
   → 用户填写当月分析（文字描述）

⑥ 月度汇总（支持下钻）
   系统按月份 + 消费标签自动汇总各分类支出（SUMIFS 逻辑）
   → 汇总类合并子标签（如"买菜"=买菜+水果，"家用"=日用品+衣服美妆+家具）
   → 点击汇总金额 → 下钻跳转到明细页面，自动筛选对应子标签

⑦ AI 分析 + 报表呈现
   用户点击"AI 分析"按钮 → 大模型分析当月消费数据 → 输出分析结论
   → 报表版面接入大模型 → 生成 HTML 页面展示月度收支汇总 + 多月对比趋势
   → 用户查看报表，完成本月记账
```

**不定期的辅助流程**：新增/调整分类规则（支持等于/包含/in 匹配）、新增消费标签、调整汇总映射、导出数据、历史数据导入（预留）。
