## 一、项目概况

### 1.1 一句话简介

将用户用了 8 年的 Excel 月度记账方式固化为 Mac 桌面应用，核心流程：**导出微信/支付宝账单 → 清洗 → 分类 → 汇总 → 报表**。

### 1.4 用户确认的关键决策（不可更改）

1. **技术栈**：Tauri 2（不用 Electron）
2. **存储**：SQLite（`~/Library/Application Support/family-accounting-app/database.db`）

8. **开发方式**：增量开发，做一块确认一块
9. **消费标签用「荭包」不是「红包」**

### Step 5-10 实际交付概要

#### Step 5：月度汇总 + 下钻跳转
- `services/summary_service.rs`：按月 GROUP BY + summary_mappings 合并 → 16 个汇总类
- `get_all_months_summary` 返回全部月份透视表（行=月，列=汇总类+手动数据）
- 下钻：点击金额 → `/transactions?month=&tags=` 自动筛选
- 表头和数据单元格悬浮显示子项构成（如"家用=日用品+衣服美妆+家具"）

#### Step 6：手动数据录入 + AG Grid
- `monthly_manual_data` 表 + `details_json` 字段存储子项分解
- 手动数据表单支持动态子项（如总资产=中行 ¥200k + 支付宝 ¥500k）
- AG Grid 替代 MUI Table：行内编辑/键盘导航/复制粘贴/排序/筛选/列拖拽/右键菜单
- 批量操作栏：标签/归属人/刚需/删除/批量采纳

#### Step 7：AI 辅助分类（DeepSeek）
- AI 模块：`ai/provider.rs`（trait）+ `ai/deepseek_provider.rs` + `ai/prompts.rs`
- 人工复核三步流程：① 自动分类 → ② AI 分类 → ③ 人工复核（采纳/修正/跳过）
- 学习机制：采纳/修正后自动生成/更新 `ai_learning_rules`
- 设置页面配置 API Key + 测试连接
- 已跳过交易管理（查看 + 恢复）

#### Step 8：AI 报表 + 图表
- AI 月度分析按钮 → 保存到 `ai_reports` 表
- AI HTML 报表生成（安全过滤 script/iframe/on*）→ WebView 渲染 + 全屏查看
- AI 图表生成：自然语言描述 → DeepSeek 返回 JSON 配置 → Recharts 渲染
- 报表历史管理

#### Step 9：规则管理 UI
- 6 个 Tab：分类规则 | AI学习规则 | 标签管理 | 汇总映射 | 规则测试 | 归属人
- 规则支持 exact/like/in 三种匹配方式
- AI 学习规则展示完整交易上下文 + 金额 + 置信度
- 归属人管理合并到规则管理页面（侧边栏减少一个入口）

#### Step 10：增强与优化
- CSV 导出、批量新增/删除、列宽 localStorage 持久化
- 右键菜单（标记刚需/排除/添加规则/复制单元格）
- AG Grid 中文 locale + 筛选器重置按钮
- 归属人下拉筛选器（自定义 floatingFilterComponent）
- 标签 Chip 着色（24 种颜色）
- 手动数据子项动态增删（Enter 确认）+ 金额精确到 2 位小数

### 3.2 分层架构约定

```
前端 (React/TS)                   后端 (Rust)
───────────────                   ────────────
pages/*.tsx    → invoke →   commands/*.rs    ← 获取 DB 连接，调用 service
api/*.ts       ← Result ←   services/*.rs    ← 业务逻辑
                             dao/*.rs         ← 纯 SQL
                             models/*.rs      ← 数据结构

⚠️ async 命令注意事项：
   MutexGuard 不能跨 .await。必须在 await 前 drop 锁：
   let (data,) = { let conn = state.conn.lock()?; (dao::query(&conn)?,) };
   // 锁已释放，安全 await
```

## 五、开发规范

### Rust
1. SQL 参数绑定用 `params![]`，禁止字符串拼接
2. 新增 .rs 文件在 `mod.rs` 注册，新增命令在 `lib.rs` 注册
3. `services/` 中部分函数因 async 命令中内联逻辑而未被调用（unused warning 可忽略）
4. ⚠️ `rusqlite` 的 `stmt.query_map().collect()` 链式调用可能导致生命周期错误，需保存到具名变量

### TypeScript / React
1. MUI **v5.18**，不要用 v6 的 API（Grid size、TextField slotProps）
2. AG Grid **Community** 版：`agSetColumnFilter`、`agAggregationComponent` 不可用
3. ⚠️ 传给 `AgGridReact` 的所有对象/函数必须用 `useMemo`/`useCallback`，新引用会导致列宽重置
4. `cellRenderer` 返回的 HTML 字符串可能被转义，用 React 组件代替

## 六、已知 Bug 和常见问题

详见 `docs/03-bug优化/Bug修复记录.md`，以下是 TOP 5：

1. **Edit 工具 replace_all**：会替换所有匹配，可能破坏代码 → 用更长的唯一上下文
2. **AG Grid props 未 memoize**：列宽选中行后还原 → useMemo/useCallback
3. **autoSizeAllColumns 失效**：localStorage 有数据时跳过 → 总是先撑满再恢复
4. **MutexGuard 跨 await**：编译报错 `!Send` → 提取数据后 drop 锁
5. **右键双弹窗**：AG Grid 内置 + 浏览器 + 自定义 → suppressContextMenu + preventDefault
