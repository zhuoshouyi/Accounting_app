# 项目记忆

## 关键决策（不可更改）

1. **技术栈**：Tauri 2（不用 Electron）
2. **存储**：SQLite，路径 `~/Library/Application Support/family-accounting-app/database.db`
3. **AI 服务商**：DeepSeek（用户提供 API Key）
4. **归属人管理**：像联系人一样只存名字，导入时选择（可跳过）
5. **AI 报表**：保存历史到 ai_reports 表
6. **月份归属**：自然归属（过 23:59:59 算第二天）
7. **开发方式**：增量开发，做一块确认一块

## 重要约定

- 消费标签用「荭包」不是「红包」
- 新增 Tauri 命令需在 `lib.rs` 的 `generate_handler![]` 注册
- 新增 .rs 文件需在 `mod.rs` 中 `pub mod xxx;` 注册
- 增量原则：在现有文件末尾追加，不修改核心逻辑
- Rust struct 用 snake_case，serde 默认输出 snake_case，TS 接口也用 snake_case

## 已知限制

1. 重复清洗风险：部分退款交易重复执行可能二次扣减，建议每批只清洗一次
2. AG Grid Community 限制：聚合状态栏、Set 筛选为企业版功能
3. 微信中性交易本期不处理

## 开发规范

- Rust：SQL 参数绑定用 `params![]`，禁止字符串拼接
- 前端：API 封装统一通过 `invokeCommand<T>()`（`src/api/client.ts`）
- 组件风格：函数组件 + hooks，MUI 为主
- AG Grid defaultColDef/filterParams/rowSelection 需用 useMemo/useCallback 保持引用稳定，否则列宽会重置
